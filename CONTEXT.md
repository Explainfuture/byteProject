# Project Context

## Glossary

### 爆款结构迁移引擎
从优质样例视频中抽取可复用的创作结构，并把结构迁移到新主题、商品信息或用户素材上的 AI 创作系统。系统目标是迁移创作方法，而不是复制样例内容。

### 样例视频
用户提供的参考短视频，用于分析脚本、镜头节奏、字幕包装、转场、BGM 卡点、封面等结构特征。

### 单视频双用模式
P0 主流程的输入方式。用户上传一条视频，系统同时把它作为样例结构学习来源和可用素材支撑度评估来源，再结合新的 Brief 生成预览方案。多条样例视频和独立素材视频上传属于增强项。

### 结构
视频中可解释、可复用、可迁移的创作组织方式。包括脚本段落结构、节奏结构、包装结构和素材槽位结构。

### 结构槽位
目标视频结构中需要被素材、文案、包装或 AIGC 内容填充的位置，例如开头吸引镜头、商品特写、使用过程、对比镜头、结尾 CTA。

### 素材缺口
用户提供的新素材无法直接满足结构槽位时产生的缺失项。素材缺口需要说明缺失内容、影响范围和建议补全策略。

### 槽位支撑度诊断
P0 的素材缺口识别方式。系统判断候选片段是否足以支撑结构槽位，输出已满足、弱匹配或缺失，并说明置信度、原因和补全策略。它不等同于真实视觉语义级的商品、人物或场景检测。

### 结构迁移
把样例视频的结构抽象映射到新内容上，生成新的脚本、分镜、时间线草案、包装建议或成片 demo。

### 包装
短视频中的非原始素材表达层，包括字幕样式、标题条、卖点卡片、贴纸、强调元素、转场、封面方案和 BGM 卡点建议。

### 预览级包装表达
P0 的包装能力。系统输出字幕样式、标题条、卖点卡片、转场、CTA 等包装建议，并在时间线或浏览器预览中表达这些包装出现的位置和意图。它不等同于生成可直接叠加到视频上的透明包装层或贴纸资源。

### 成片 demo
用于评审验证系统闭环的可播放或可交互预览输出。P0 阶段可以是基于时间线、素材说明、字幕卡、包装层和镜头建议生成的低保真浏览器预览，不等同于服务端导出的 MP4 文件。

### 爆款质量 Benchmark
系统对每个生成候选执行的 100 分制质量评测。它评估生成视频是否有吸引力、是否按用户需求改写文案、是否迁移了样例/历史优秀案例的有效结构、是否有清晰节奏和可观看包装，以及是否满足安全和可解释要求。它不是真实平台 CTR、GMV 或推荐量预测。

### BenchmarkScore
爆款质量 Benchmark 的结构化输出。它包含总分、7 个维度的分数和证据、硬性失败规则、top fixes、是否 accepted，以及可直接传给 Model Adapter 的 `revisionBrief`。

### 评测驱动重新生成
生成候选低于阈值时的自动迭代方式。低于 60 分或触发硬性失败必须重生成；60-79 分可以继续优化；80 分及以上且无硬性失败才能作为 accepted candidate。重新生成必须沿用同一轮样例结构、素材槽位、用户事实和安全边界，只重写脚本、时间线、包装、转场、节奏提示和渲染提示。

### 营销类商品短视频
P0 主演示场景。围绕商品卖点、痛点 hook、商品亮相、使用场景、对比证明、优惠或 CTA 生成短视频方案。

### 槽位匹配
把结构槽位需要的素材类型、表达意图和时长，与用户提供的图片、视频或文案素材进行匹配的过程。匹配结果包括已满足、弱匹配和缺失。

### 低保真成片 demo
P0 可播放验证结果。优先使用 Remotion Player、HyperFrames 风格描述或 HTML timeline preview 根据时间线渲染，由用户素材信息、字幕卡、卖点卡、简单转场、局部放大建议和重复利用建议组合而成，不追求商业级最终画质，也不强制要求导出 MP4。当前主路径生成 10 个本地风格赛道预览，每个预览目标时长不超过 60 秒。

### Mock 降级
当真实模型、ASR 或视频分析不可用时，系统使用内置样例分析、规则结果或手动输入继续跑通演示闭环。

### 关键帧级轻量理解
P0 的视频理解范围。系统从上传视频中抽取少量关键帧，结合视频元数据、用户 Brief、可用模型和本地规则推断结构、节奏、包装和素材槽位。它不等同于完整 ASR、逐帧视觉理解或精准镜头切分。

### 结构知识库
系统沉淀的可复用剪辑经验集合。包括样例拆解出的脚本结构、节奏模式、素材槽位、包装建议、缺口补全策略和适用场景。

### 会话级结构知识库
P0 的知识库形态。系统内置种子原子，并在当前运行会话内新增和引用样例拆解结果。它不承诺跨重启持久化，也不等同于数据库或长期知识库。

### 长视频素材
评测和真实使用中更常见的新内容输入形式。用户上传一段较长视频，系统按中等预算抽取 4-16 张关键帧、粗切、筛选候选片段，并按结构槽位重构为默认 10-20 秒、最长 60 秒内的新短视频。

### 风格赛道预览
P0 的本地成片展示方式。系统基于同一脚本、分镜和时间线生成 10 个可播放或可交互的 Remotion/HyperFrames 风格赛道预览，用于展示不同垂类落地方式，例如电商爆品、测评对比、B 端营销、口播知识、生活 Vlog、MG 信息流、活动促销、教程步骤、品牌质感和剪辑卡点。

### 评测替换素材
评委可能使用自己的视频素材测试系统，而不是只观看提交时绑定的固定 case。系统主流程必须能接受替换视频并重新完成分析、匹配、缺口识别和生成。

### 原子技巧
从样例视频中拆出的最小可复用创作方法，例如痛点 hook、商品亮相、节奏加速、卖点三连、对比证明、标题条补位、强 CTA、卡点转场。系统迁移的是这些原子技巧的组合方式，而不是完整复制样例模板。

### Composition Plan
系统在生成新视频前形成的组合计划。它说明选用了哪些原子技巧、如何映射到结构槽位、哪些素材被使用、哪些缺口被补全，以及为什么这样编排。

### 轻量对话式重新生成
P0 可选的人机协同方式。用户用一句话补充修改意图，系统把该指令追加到下一轮 Brief 中重新生成整体方案。它不是精确局部改片，不承诺只更新某一个镜头或稳定保留全部时间线标识。

### 策略化单版本生成
P0 的版本能力。用户选择高点击、高转化、高节奏、高质感等创作策略后，系统按该策略生成一个主方案。它不同于多版本并列生成，后者要求同一输入一次性产出多个完整方案并支持对比。

### 透明包装层
由 Remotion 或类似框架生成的透明 Alpha 视频片段，例如字幕卡、标题条、卖点卡片、MG 动效和贴纸强调层。它可以通过 FFmpeg 叠加到用户素材视频上，形成最终成片；该能力属于 P1/增强项，不是 P0 的硬承诺。

### AIGC 补全接口预留
P1/加分项能力。系统可以保留静态封面、背景、配音或补充画面的生成接口和策略枚举，但 P0 不宣传真实 AIGC 补全能力；P0 的可执行补全以文案/字幕、包装卡片和现有素材重组复用为主。

### 产品化错误
面向用户和评审展示的脱敏错误。它说明模型是否可用、是否降级、失败属于鉴权/网络/配置/抽帧等哪类问题，但不暴露 provider 原始响应、堆栈、本地路径、API key 或 endpoint id。

### Agent 化工具编排体验
P0 的 Agent 形态。系统按固定工具路径组织上传检查、样例分析、知识检索、素材评估、方案生成和预览渲染，并展示 trace 和 fallback 状态。模型可用时可参与工具调用和创意增强；模型不可用时由确定性 workflow 跑通主链路。它不等同于完全自主 Agent。

### 可替换模型适配器
模型调用的边界。系统通过稳定的 Model Adapter 协议接入多模态理解或语言模型，provider 可以从 Ark/Doubao-compatible 接口替换为其他模型，但核心结构协议、槽位匹配协议、缺口补全协议和前端展示数据结构不应随模型替换而改变。

### 视频产物 case
比赛交付材料中的可观看视频结果。它可以由系统预览录屏、手动导出预览或后续增强脚本生成，不等同于 P0 系统必须在线导出的 MP4 文件。
