# UI Guidelines: 爆款结构迁移引擎

## Design Direction

界面定位为“创作结构控制台”，不是营销落地页。用户打开后第一眼应看到可操作的工作台：单视频上传、关键帧抽取、结构拆解、缺口映射、时间线结果和 demo 预览。

视觉方向采用克制但有辨识度的生产工具风格：

- 基底：温和浅色工作区，搭配深色检查器/时间线区域。
- 强调色：用少量高对比色标记结构、缺口、匹配状态和生成动作。
- 质感：清晰边界、细线、分层阴影、数据标记、时间线刻度、结构标签。
- 气质：专业、密集、可扫描，像剪辑软件和数据分析工具的结合，而不是普通 SaaS 首页。

避免：

- 过度居中布局。
- 大面积紫色渐变。
- 只有卡片堆叠的静态仪表盘。
- 过圆的圆角和低信息密度。
- 只用通用字体、通用蓝色按钮、通用空状态。
- 用大段说明文字解释功能。

## Product Layout

P0 工作台建议采用五区布局：

1. 顶部任务栏
   - 项目名、case 切换、mock/real 模式状态、导出按钮、生成按钮。

2. 左侧输入区
   - 单个样例视频上传。
   - 关键帧抽取状态。
   - 新主题、商品卖点、目标人群。
   - 人工校正入口。

3. 中央结构画布
   - 样例结构分层展示：脚本结构、节奏结构、素材槽位结构。
   - 槽位节点按 segment 排列：hook、body、proof、offer、cta。
   - 用连线或映射列展示样例槽位如何迁移到新内容。

4. 右侧检查器
   - 当前槽位详情。
   - 匹配素材。
   - 缺口原因。
   - 补全策略。
   - 人工调整控件。

5. 底部时间线
   - 时间码、镜头块、字幕、包装卡片、转场和节奏标记。
   - 时间线需要稳定尺寸，不因 hover 或文本变化跳动。

## Component Rules

- 按钮使用图标 + 简短标签，常见动作优先用 lucide 图标。
- 工具栏按钮保持固定尺寸，避免 hover 后布局抖动。
- 状态标签使用稳定颜色：
  - matched：低饱和靛紫或深灰。
  - weak_match：琥珀色。
  - missing：红色或珊瑚色。
  - generated：靛色或黑色。
- 卡片只用于重复实体，例如素材、槽位、缺口、版本；页面大区不要再包一层大卡片。
- 表单控件优先使用 segmented controls、toggle、slider、select、stepper，不用大段说明文字。
- 所有长文本必须可换行、可截断或可展开，不能溢出按钮、卡片和时间线块。
- 时间线、视频预览、槽位图必须有固定比例或固定轨道高度。

## Interaction Requirements

- 用户应能一眼看出四件事：
  - 上传视频抽取了什么结构。
  - 这些结构如何迁移到新的主题和商品信息。
  - 哪些槽位缺失。
  - 系统如何补全并生成结果。
- 点击任意槽位后，右侧检查器同步显示结构意图、素材匹配和补全策略。
- 点击任意缺口后，时间线高亮受影响片段。
- 调整 hook、卖点顺序、节奏或 CTA 后，生成结果必须有可见变化。
- mock 模式和真实模型模式要在界面中明确标记，但不要打断演示流程。

## Visual Verification

前端完成后必须做浏览器验证：

- 启动本地 dev server。
- 使用浏览器打开实际页面。
- 检查桌面和移动宽度下是否有重叠、溢出、空白画布或按钮文本挤压。
- 验证上传、切换 case、点击槽位、查看缺口、生成结果、导出入口这些主路径。
- 截图检查首屏是否是实际工作台，而不是 landing page。

## Skill Usage Notes

已安装并可用于后续 UI 工作的相关 skills：

- `frontend-design`：用于创建高质量、非模板化的 React/Web 界面。
- `web-artifacts-builder`：用于复杂 React + TypeScript + Tailwind + shadcn/ui artifact 工作流参考。
- `webapp-testing`：用于 Playwright 浏览器验证。
- `theme-factory`：用于需要主题化时选择或生成一致的颜色和字体系统。
